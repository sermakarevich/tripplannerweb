# Website

Static marketing site — no build step, no framework.

Pages:
- `index.html` — landing: hero, how it works, why different, CTA band.
- `design_my_trip.html` — "Design my trip": full trip brief as a numbered route-style form,
  pre-filled with the Adriatic 2026 example. On submit it (1) POSTs to FormSubmit.co AJAX
  (sermakarevich@gmail.com — first-ever submission triggers a one-time activation email that
  must be confirmed) and (2) best-effort POSTs the brief to the portal's public
  `POST /api/wishlist` route, which persists it to the `WishlistBrief` Postgres table.
  Set the `WISHLIST_API` constant in each page's script to `https://<portal-domain>/api/wishlist`
  to enable DB persistence (empty = email only). The submit button is hard-disabled during the
  request and guarded against rapid multi-click. Button = "Join wishlist".
- `eurotrip.html` — the real trip page generated by the agent: self-contained (own inline
  styles), site header nav on top, and a tab bar with three actions — **Design my trip**
  (links to the pre-filled brief), **Planner**, **Places**. This is the template shape for
  per-traveler pages we generate and place at hidden URLs.
- `styles.css` — shared design system (editorial travel-journal look, matches trip artifacts).

Serve locally:

```
python3 -m http.server 8888 --directory applications/website
```

Deploy: any static host (Firebase Hosting / Cloudflare Pages).
